# Onboard New Developer

## Overview
Comprehensive onboarding process to get a new developer up and running quickly.  

## Steps
1. **Environment setup**
   - Install required tools (language runtimes, package managers, Docker, etc.)
   - Set up development environment and editor configuration
   - Configure git, SSH keys, and access to repositories
   - Set up environment variables and secrets management

2. **Project familiarization**
   - Review project structure and key directories
   - Understand architecture and main data flows
   - Read key documentation (README, CONTRIBUTING, ADRs)
   - Set up local databases, queues, or external services as needed

3. **Hands-on start**
   - Run the application locally end‑to‑end
   - Run the test suite and ensure it passes locally
   - Pair on a small, low‑risk task or bug fix
   - Walk through code review expectations and PR process

## Onboarding Checklist
- [ ] Development environment ready
- [ ] Can run application locally
- [ ] All tests passing locally
- [ ] Access to required services and dashboards
- [ ] Completed first PR and review cycle
